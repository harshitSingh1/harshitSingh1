# Design Inspiration

> Principles borrowed — not websites copied.

---

## Sources

### Apple Documentation
- **Taken:** Clear typographic hierarchy, generous whitespace, spec-sheet field layouts, confidence through restraint.
- **Rejected:** Marketing photography, product renders, animated scroll.

### Linear
- **Taken:** Monochrome palette, precise borders, calm editorial rhythm, numbered changelog structure.
- **Rejected:** App UI chrome, sidebar navigation patterns.

### Vercel
- **Taken:** Builder-first tone, monospace for technical metadata, high information clarity.
- **Rejected:** Geometric marketing hero, black-on-white landing page layout.

### Anthropic
- **Taken:** Human warmth within technical content, thoughtful pacing, trustworthy tone.
- **Rejected:** Serif display typography (unavailable on GitHub).

### GitHub
- **Taken:** Native markdown rendering, system fonts, link styling, contribution graph as earned credential.
- **Rejected:** Profile README trope patterns (shields, trophies, typing SVG).

### RFC Documents
- **Taken:** Numbered sections, formal structure, metadata headers, revision tracking.
- **Rejected:** Dense prose blocks, ASCII diagrams.

### Architectural Blueprints
- **Taken:** Grid systems, registration marks, title blocks, revision tables, field labels.
- **Rejected:** Full technical drawing complexity, dimension lines.

### Engineering Specification Sheets
- **Taken:** Label-value pairs, part numbers, status fields, operator/author attribution.
- **Rejected:** Multi-page pagination, stamp overlays.

### Patent Filings
- **Taken:** Document ID format, inventor/operator field, classification codes for domains.
- **Rejected:** Legal boilerplate, figure numbering.

### Construction Drawings
- **Taken:** Scale indicators, sheet numbers, cross-reference notation.
- **Rejected:** Isometric views, material callouts.

---

## Synthesis

The profile reads as **Sheet 01 of an Engineering System Specification** — a document that happens to live on GitHub, not a GitHub profile that pretends to be a document.

The blueprint grid provides visual identity without decoration. The spec fields provide structure without widgets. The content provides proof without boasting.
