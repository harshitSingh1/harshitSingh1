# Engineering Capabilities — Redesign

Grouped by **what you can build**, not by language layer.

---

## Design Goal

Replace the traditional `Languages / Frontend / Backend` wall with capability domains that reflect engineering maturity.

---

## Layout

Section header → capability blocks stacked vertically.

Each block:
```
##### AI Engineering
`Python` · `LangGraph` · `LangChain` · `RAG` · `FastAPI` · `Ollama` · `PyTorch` · `TensorFlow` · `Vertex AI` · `Gemini` · `OpenAI`
```

Rules:
- Capability name as `#####` heading
- Technologies as single inline mono line
- Max 12 technologies per block — ` · ` separated
- No icons, no badges, no progress bars
- Blank line between blocks
